# 0027 — Visual session timeline (the forest, rendered)

## Status: IMPLEMENTED (2026-07-06, feature/0027-visual-timeline — phases a–d).
## Phase e (diff panel + replay-divergence badges + richer filters) pending.
## Companion to 0026 (interactive time travel, merged).

## Motivation

Every session relationship arc can create is already recorded in `meta.json`:

| stamp | written by | relationship |
|---|---|---|
| `resumed_from` | resume (mode 1) | continued from the end |
| `resumed_from` + `branched_at_turn` | resume `--at-turn` (mode 4), 0026 `/rewind` | forked at turn N |
| `replay_of` + `replay_mode` | replay (modes 2/3) | verification re-run |
| `rerun_of` | rerun (mode 5) | fresh agent, same inputs |
| `retry_of_turn` | 0026 `/retry` | same prompt, re-rolled |
| `provider_override` | 0026 `/model` | model swap on branch |

The forest is fully reconstructable today — and completely invisible. You
operate the five replay modes blind, via session ids and turn numbers.
`arc compare` gives text metrics for sessions you already know about; nothing
shows you *what exists* and *how it relates*.

The timeline is the projection that makes arc's thesis visible: sessions as
a navigable tree of experiments — turns as nodes, branches as forks, cost
and divergence as overlays. Once 0026 ships, forks become an everyday
artifact; this is the screen where you see which branch cracked the binary
and what each attempt cost.

## Decisions already made (with the user)

- **Static, self-contained HTML + vanilla JS.** No server, no CDN, no
  dependencies — CSS/JS inline, data embedded as JSON. Honors arc's
  zero-dep ethos; a browser is simply the right tool for a 2D graph with
  pan/click/expand, which inline prompt_toolkit is not.
- **Artifacts live in the sessions dir, alongside the logs.** They are
  session artifacts like events.jsonl and session.log, regenerated as
  sessions are created/resumed/branched — principle 2: everything rebuilds
  from events + meta; this is just another projection.

## File layout: two tiers

Browsers block `fetch()` from `file://` pages (CORS), so a static page can
only use what's embedded in it or reachable by a plain `<a href>`. Hence:

```
sessions/
  timeline.html          the forest — every session, lineage edges, summaries
  <sid>/
    session.html         per-session detail — full turn-by-turn render
    events.jsonl …       (existing artifacts, unchanged)
```

- **`timeline.html`** embeds the forest model + per-turn *summaries*
  (truncated text). Rebuilt whenever any session ends — it's the global view.
- **`<sid>/session.html`** embeds that session's full detail (complete tool
  outputs, thinking blocks, per-call tokens). Written once at session end;
  immutable afterwards, like the recording it renders. Timeline nodes link
  to it with plain anchors (`<sid>/session.html#turn-3`), which work from
  `file://`.

This split keeps timeline.html small at hundreds of sessions (summaries
only) while full fidelity stays one click away, and it matches artifact
lifecycle: global file mutable, per-session files write-once.

## The model

```
src/arc/timeline/
  scan.py      sessions dir → Forest (meta.json pass; index.jsonl for order)
  model.py     Forest / SessionNode / TurnNode / Edge dataclasses
  summarize.py events.jsonl → per-turn summaries (reuses replay/compare.py
               extraction; truncation limit from config)
  render.py    Forest → timeline.html; session → session.html (stdlib
               string.Template over an inline-asset template)
  assets/      template.html, timeline.css, timeline.js (inlined at render)
```

- **SessionNode**: sid, created_at, provider/model, turn count, totals
  (tokens, cost via the compare summarizer), status (completed/aborted),
  lineage stamps, `is_subagent` (sub-agent sessions render as collapsed
  child lanes attached to their parent's dispatching turn).
- **TurnNode**: index, user-input summary, assistant summary, tool-call
  count, tokens/cost/duration.
- **Edge**: `(parent_sid, parent_turn | None, child_sid, kind)` where kind ∈
  `branch | resume | retry | replay | rerun`. Kind is derived from the meta
  stamps table above; `parent_turn` is `branched_at_turn` for
  branch/retry, None (attach at end) for resume, None (attach at lane
  head, dashed) for replay/rerun.
- Orphans (parent session deleted/wiped) render as roots with a "parent
  missing" badge — never an error.

## The page

Layout: horizontal lanes, one per session, grouped by tree; time flows
left→right, turns are square nodes on the lane. Fork edges drop from parent
turn N to the child lane head. Edge kinds are visually distinct (solid =
branch/retry, dotted = resume, dashed gray = replay/rerun). Pure
SVG generated by timeline.js from the embedded JSON — no canvas, no
library. Pan/zoom = scroll + CSS transform.

Interactions (v1):

- **Click a turn node** → detail panel: full-ish summaries, tokens, cost,
  duration, tool-call list; link to `<sid>/session.html#turn-N`.
- **Branch from here** → shows `arc resume <sid> --at-turn N` (and, post-
  0026, the `/rewind` equivalent) with a copy button. A static page cannot
  execute it; copyable command is the honest v1. (`--serve` mode that runs
  it is a possible phase 2, stdlib `http.server`.)
- **Select two sessions → diff** → side-by-side turn columns, first
  textual divergence highlighted, metric deltas (tokens/cost/turns) in a
  header row. Client-side over embedded summaries — this is `arc compare`'s
  data model rendered visually. Deep full-text diff stays in the terminal
  (`arc compare`); the panel links the command.
- **Replay-lane divergence marker**: for `replay_of` sessions, the first
  turn whose summary differs from the source gets a ⚠ badge (computed at
  render time, not in JS — the scanner has both sessions).
- Filter box: by text, provider/model, date; collapse finished trees.

Deliberately not in v1: live-updating view of a running session, executing
commands from the page, editing/annotation, cross-tree search beyond the
filter box.

## Regeneration: builtin plugin + CLI

**`timeline` builtin plugin** (the default path — keeps artifacts fresh
without the user thinking about it):

- `on_session_end`: write this session's `session.html`, then rescan metas
  and rewrite `timeline.html`.
- Cost control: the forest pass reads only `meta.json` + `index.jsonl`
  (cheap at hundreds of sessions). Turn summaries for *other* sessions are
  not recomputed — each session's summaries are cached as
  `<sid>/timeline.node.json` at its own session end, so a rebuild is
  meta-scan + read N small JSONs + template. The only events.jsonl parsed
  per rebuild is the session that just ended.
- Failure is quarantined like any plugin; a broken timeline never touches a
  session.

**`arc timeline [--open] [--rebuild]`** CLI:

- No args: ensure timeline.html is current (rebuild if missing/stale),
  print its path. `--open`: also `webbrowser.open()` it. `--rebuild`:
  force-regenerate everything including per-session files (recovery after
  format changes or `arc wipe --sessions` remnants).
- Also surfaces as a line in the `arc replay` hub menu ("open timeline").

Config (`defaults.py`, under `plugins.enabled.timeline.config`):

```yaml
timeline:
  enabled: true
  summary_max_chars: 400        # per-field truncation in embedded summaries
  full_output_max_chars: 20000  # per tool output in session.html
```

`arc wipe --sessions` already removes the dir wholesale; the plugin
regenerates from whatever remains. No new wipe surface needed.

## Rendering fidelity notes

- All embedded text is HTML-escaped and JSON is embedded via
  `<script type="application/json">` (not string-concatenated JS) — tool
  outputs contain arbitrary bytes/HTML; treat them as hostile.
- Binary/base64 metadata blocks render as `[binary: N bytes]`.
- Theme: single theme in v1, styled to match the arc brand palette; a
  `prefers-color-scheme` media query covers light/dark. Mapping arc's TUI
  themes onto the page is polish, not v1.
- session.html for a 100k-token session with `full_output_max_chars`
  truncation lands ~1–2 MB — acceptable for file:// pages. Truncated
  outputs end with `… (truncated; full output: arc log <sid>)`.
